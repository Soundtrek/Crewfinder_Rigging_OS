# AI Boundaries

## Purpose
Define strict limits to prevent unsafe or misleading use of AI.

---

## Absolute Boundaries
AI must never:
- approve a rigging setup
- replace engineering judgement
- override system rules
- act without user initiation

---

## System Protection
The system must ensure:
- AI cannot modify stored calculations
- AI cannot modify job state
- AI cannot trigger signoff

---

## Communication Rule
AI must always be presented as:
- advisory
- optional
- secondary to system results

---

## Failure Mode Rule
If AI fails or is unavailable:
- system must continue to operate fully
- no core feature must depend on AI

---

## Final Rule
AI enhances understanding.
It does not control outcomes.
